## 特殊字符导致认证失败（根因定位）

- Script Key `rsdebbv^josz2zAiwltdgpjyd` 中含 `^` 字符
- `^` 是 Windows CMD 的转义字符，通过命令行参数 `--script-key` 传递给外部进程时可能被吞掉
- PowerShell → `vx.exe` → `cargo.exe` → `fpt-cli.exe` 的链路上经过多层进程创建，`^` 风险极高
- CLI 的 `resolve()` 逻辑是 `overrides.script_key.or(env_script_key)` → 命令行参数优先，被截断的值会覆盖环境变量中正确的值

### 修复措施
1. `.env` 中含特殊字符的值用双引号包裹
2. `Build-AuthArgs` 不再传凭据命令行参数（`--script-key`、`--password` 等），只传 `--site` 和 `--auth-mode`
3. 凭据全部通过环境变量传递（进程继承，不经过命令行解析，特殊字符安全）
4. 新增特殊字符检测诊断输出
